# 墨刀原型 MCP 使用说明

本文档给使用者说明如何启动和使用 `modao-prototype-mcp`，把墨刀 read-only 原型链接导出为页面目录和截图。

> 重要说明：如果使用远程 MCP，例如 `http://203.0.113.10:3001/mcp`，客户端只需要配置远程 URL，不需要也不能配置服务器上的 `dist/index.js` 路径。
>
> 另外，工具参数里的 `outputDir` 是 MCP 服务器上的保存目录，不是使用者自己电脑上的目录。比如远程服务运行在 `203.0.113.10` 上，那么导出的文件会保存在这台服务器的 `outputDir` 中。

## 这个工具能做什么

`modao-prototype-mcp` 是一个 MCP Server，用来读取墨刀只读分享链接，并自动导出：

- `manifest.json`：页面清单和图片路径
- `catalog.md`：方便阅读的页面目录和图片预览
- `result.json`：机器可读结果
- `images/**/*.png`：每个原型页面的截图；如果识别到墨刀左侧目录，会按目录分组保存

它还支持对已经导出的目录进行图片补全或刷新。

## 准备工作

进入项目目录：

```bash
cd /root/book/modao-prototype-mcp
```

安装依赖并构建：

```bash
npm install
npm run build
```

如果机器没有 Chrome，需要安装 Playwright 浏览器：

```bash
npx playwright install chromium
```

## 推荐方式：连接远程 MCP 下载服务

远程 MCP 不能直接写入使用者电脑上的任意路径。下载模式的流程是：

1. 远程服务器导出墨刀原型
2. 服务器自动打包成 zip
3. MCP 返回 `zipUrl`、`catalogUrl`、`manifestUrl`、`resultUrl`
4. 使用者用浏览器下载到自己电脑

使用者客户端只配置远程 URL，不配置服务器上的 `dist/index.js` 路径：

```json
{
  "mcpServers": {
    "modao-prototype-remote": {
      "url": "http://203.0.113.10:3001/mcp",
      "headers": {
        "Authorization": "Bearer 替换成你的密钥"
      }
    }
  }
}
```

远程服务地址：

```text
http://203.0.113.10:3001/mcp
```

健康检查：

```text
http://203.0.113.10:3001/health
```

### 推荐工具：export_modao_prototype_download

远程使用时优先调用：

```text
export_modao_prototype_download
```

输入示例：

```json
{
  "url": "https://modao.cc/proto/xxx/sharing?view_mode=read_only&screen=xxx",
  "name": "招聘系统原型",
  "headless": true,
  "timeoutMs": 45000
}
```

注意：这个工具没有 `outputDir` 参数，因为远程服务无法直接写入使用者电脑路径。

返回结果会包含：

```json
{
  "exportId": "zhao-pin-xi-tong-yuan-xing-20260622-102030",
  "zipUrl": "http://203.0.113.10:3001/download/zhao-pin-xi-tong-yuan-xing-20260622-102030.zip",
  "catalogUrl": "http://203.0.113.10:3001/download/zhao-pin-xi-tong-yuan-xing-20260622-102030/catalog.md",
  "manifestUrl": "http://203.0.113.10:3001/download/zhao-pin-xi-tong-yuan-xing-20260622-102030/manifest.json",
  "resultUrl": "http://203.0.113.10:3001/download/zhao-pin-xi-tong-yuan-xing-20260622-102030/result.json"
}
```

使用者打开 `zipUrl`，就可以把完整导出包下载到自己的电脑。

### 旧工具说明：export_modao_prototype

旧工具 `export_modao_prototype` 仍然保留，但它的 `outputDir` 是服务器路径，不是使用者电脑路径。远程用户一般不要用它，除非明确知道服务器文件系统路径。

## 方式一：本地 stdio MCP 使用

如果是在同一台机器上本地运行 MCP，才使用下面这种 `command` 配置：

```json
{
  "mcpServers": {
    "modao-prototype": {
      "command": "node",
      "args": ["/root/book/modao-prototype-mcp/dist/index.js"]
    }
  }
}
```

配置完成后，客户端会出现两个工具：

| 工具名 | 用途 |
| --- | --- |
| `export_modao_prototype` | 导出墨刀原型目录和截图 |
| `update_modao_export_images` | 更新已有导出目录里的截图 |

### 导出原型

调用工具：`export_modao_prototype`

示例输入：

```json
{
  "url": "https://modao.cc/proto/xxx/sharing?view_mode=read_only&screen=xxx",
  "outputDir": "/root/book/modao-prototype-mcp/exports/my-prototype",
  "headless": true,
  "timeoutMs": 45000
}
```

常用参数说明：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `url` | 是 | 墨刀 read-only 分享链接 |
| `outputDir` | 是 | 导出结果保存目录 |
| `headless` | 否 | 是否无头运行浏览器，默认 `true` |
| `timeoutMs` | 否 | 页面加载超时时间，默认 45000 毫秒 |
| `startDirectory` | 否 | 从第几个左侧目录开始导出，适合分批导出 |
| `maxDirectories` | 否 | 最多导出多少个目录 |

导出完成后，重点查看：

```text
exports/my-prototype/catalog.md
```

### 更新已导出的图片

调用工具：`update_modao_export_images`

只补缺失或损坏图片：

```json
{
  "outputDir": "/root/book/modao-prototype-mcp/exports/my-prototype",
  "mode": "missing",
  "headless": true
}
```

全量刷新所有图片：

```json
{
  "outputDir": "/root/book/modao-prototype-mcp/exports/my-prototype",
  "mode": "all",
  "headless": true
}
```

更新完成后会生成：

```text
update-report.json
```

里面会记录每张图片是更新、跳过还是失败。

## 方式二：命令行直接使用

如果不通过 MCP 客户端，也可以直接使用 CLI。

### 导出完整原型

```bash
npm run export -- "https://modao.cc/proto/xxx/sharing?view_mode=read_only&screen=xxx" exports/my-prototype
```

### 显示浏览器窗口调试

如果遇到登录、验证码、加载失败等问题，可以加 `--headed`：

```bash
npm run export -- "https://modao.cc/proto/xxx/sharing?view_mode=read_only&screen=xxx" exports/my-prototype --headed
```

### 分批导出

如果原型页面很多，一次导出太慢，可以按左侧目录分批：

```bash
npm run export -- "https://modao.cc/proto/xxx/sharing?view_mode=read_only&screen=xxx" exports/batch-01-06 --start-directory 1 --max-directories 6

npm run export -- "https://modao.cc/proto/xxx/sharing?view_mode=read_only&screen=xxx" exports/batch-07-12 --start-directory 7 --max-directories 6
```

### 更新已有导出

只补缺失或损坏图片：

```bash
npm run update -- exports/my-prototype
```

全量刷新：

```bash
npm run update -- exports/my-prototype --mode all
```

强制刷新：

```bash
npm run update -- exports/my-prototype --force
```

## 方式三：启动远程 MCP 服务

适合部署到服务器，让支持远程 MCP 的客户端连接。

启动服务：

```bash
HOST=0.0.0.0 PORT=3000 MCP_TOKEN="替换成你的密钥" npm run start:http
```

接口地址：

| 地址 | 用途 |
| --- | --- |
| `GET /health` | 健康检查 |
| `/mcp` | Streamable HTTP MCP 端点 |
| `/sse` | SSE MCP 端点 |
| `/messages` | SSE 消息投递端点 |

远程 MCP 客户端配置示例：

```json
{
  "mcpServers": {
    "modao-prototype-remote": {
      "url": "http://203.0.113.10:3001/mcp",
      "headers": {
        "Authorization": "Bearer 替换成你的密钥"
      }
    }
  }
}
```

如果部署到公网，建议：

1. 必须设置 `MCP_TOKEN`
2. 前面加 Nginx、Caddy 或 Cloudflare 做 HTTPS 反代
3. 不要把无鉴权服务直接暴露到公网

## 导出结果怎么看

导出目录结构大致如下：

```text
exports/my-prototype/
├── catalog.md
├── manifest.json
├── result.json
└── images/
    ├── 001-首页/
    │   └── 001-页面-1-xxx.png
    ├── 002-订单管理/
    │   └── 001-页面-1-xxx.png
    └── ...
```

如果页面没有暴露左侧目录，截图可能会直接保存为 `images/*.png` 的平铺结构。

推荐先打开：

```text
catalog.md
```

它包含：

- 原型源链接
- 导出时间
- 页面数量
- 页面分组目录
- 每张截图预览

如果要给程序读取，使用：

```text
manifest.json
```

## 常见问题

### 1. 导出很慢怎么办？

使用分批参数：

```bash
--start-directory 1 --max-directories 6
```

一次只导出几个左侧目录。

### 2. 截图缺失或损坏怎么办？

使用更新命令补图：

```bash
npm run update -- exports/my-prototype
```

### 3. 页面需要登录或验证码怎么办？

用有头模式调试：

```bash
npm run export -- "墨刀链接" exports/my-prototype --headed
```

### 4. 输出目录里最重要的是哪个文件？

给人看用：

```text
catalog.md
```

给程序或 AI 读取用：

```text
manifest.json
```

### 5. 链接有什么要求？

必须是 `modao.cc` 的 read-only 分享链接，例如：

```text
https://modao.cc/proto/xxx/sharing?view_mode=read_only&screen=xxx
```
